Word prediction part1

The goal of the project is to build a model that can predict the next word given an input word/sentence fragment. This report demonstrates getting and loadeding of data, basic report of summary statistics about the data sets, some exploratory analysis, reporting interesting findings and the plans for the eventual app and prediction algorithm.

Data processing

cleaning corpus and removing profanity words

library(tm)
docs <- tm_map(docs, content_transformer(tolower))
docs <- tm_map(docs, removePunctuation)
docs <- tm_map(docs, removeNumbers)
docs <- tm_map(docs, removeWords, stopwords("english"))   
docs <- tm_map(docs, stemDocument)
docs <- tm_map(docs, stripWhitespace)
docs <- tm_map(docs, PlainTextDocument)
conprofane <- file("C://texts/profanity.txt", "r")
profanity_vector <- VectorSource(readLines(conprofane))
docs <- tm_map(docs, removeWords, profanity_vector)

Plans for Prediction Algorithm and Shiny App

The prediction algorithm will be based on the data used for this report.The data is cleaned to be lower case, without punctuation, numbers, or white space, stop and profanity words. The predictor will take the entered word or phrase (up to three previous words) and search the term document matrix of data for the most frequently used words following the queried word or phrase. It will then return the top five words ranked based on frequency. The Shiny app will utilize this predictor by having an input word so that a word or phrase may be entered and there will be five returned values as popular options for the next word in the phrase in order of likelihood.
